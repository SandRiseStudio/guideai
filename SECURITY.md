# Security Policy

## Reporting a Vulnerability

Please **do not report security vulnerabilities through public GitHub issues, discussions, or pull requests**.

Instead, report vulnerabilities privately to the project maintainers using a secure reporting channel established by the GuideAI team.

If you are working inside the GuideAI organization, use the approved internal security reporting path. If you are an external reporter and no private channel has been published yet, contact the maintainers through the repository owner contact listed in the project profile and request a secure disclosure path.

## What to Include

Please include as much of the following as possible:

- affected component or file path
- reproduction steps
- proof of concept, if available
- impact assessment
- suggested remediation, if known
- whether credentials, secrets, tokens, or customer data may be involved

## Response Expectations

The maintainers will aim to:

- acknowledge receipt of the report promptly
- investigate and validate the issue
- determine severity and remediation plan
- coordinate a fix and disclosure timeline when appropriate

## Secrets Handling

If you discover leaked credentials or secrets:

1. avoid reposting the secret in issues or pull requests
2. notify maintainers through the private security channel
3. rotate impacted credentials immediately if you control them
4. remove secrets from code, logs, and artifacts

## Supported Versions

Security fixes are handled on a best-effort basis for actively maintained branches and releases.

## Scope

This policy applies to the open-source repository contents. Enterprise-only deployments, managed environments, and internal systems may follow additional private security procedures.
